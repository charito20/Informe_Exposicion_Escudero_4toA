# GRUPO H: rmToo

**Integrantes:** Alexander Tigasi, David Vaca y Fabiola Huilcapi.

## Alexander Tigasi

### Teoría
- Introdujo rmToo como una herramienta basada en el concepto de **"Requirements as Code"**.
- Explicó que utiliza archivos de texto plano para gestionar requisitos, separando el contenido del formato.
- Comparó este enfoque con los documentos tradicionales, destacando un mejor control de versiones y trazabilidad.
- Mencionó que rmToo aprovecha herramientas como Git, Python, LaTeX y Graphviz para automatizar la documentación.

### Práctica
- Presentó la estructura general de la herramienta y el flujo de trabajo basado en archivos de requisitos.

## David Vaca

### Teoría
- Explicó el funcionamiento interno de rmToo mediante un flujo de procesamiento de requisitos.
- Describió que la herramienta genera automáticamente documentos PDF, HTML y diagramas de trazabilidad.
- Comparó rmToo con herramientas como IBM DOORS, Jama y asistentes de inteligencia artificial.
- Destacó como ventajas el uso de Git, la integración con CI/CD y la automatización, además de mencionar como limitación la ausencia de una interfaz gráfica.

### Práctica
- Mostró el proceso de generación de artefactos y documentación a partir de archivos de texto.
- Explicó el flujo de validación y trazabilidad de los requisitos.

## Fabiola Huilcapi

### Teoría
- Presentó investigaciones recientes sobre la automatización en Ingeniería de Requisitos.
- Explicó el concepto de Living Documentation o documentación viva.
- Destacó que la trazabilidad contribuye a reducir errores y mejorar la consistencia de los proyectos.
- Señaló que rmToo implementa estas prácticas de forma nativa mediante la integración con el repositorio del proyecto.

### Práctica
- Realizó una demostración del funcionamiento de la herramienta.
- Mostró cómo los requisitos pueden mantenerse actualizados y sincronizados automáticamente mediante la generación de documentación.
